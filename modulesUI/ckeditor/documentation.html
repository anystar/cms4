CKEditor Documentation

Quick use

Wrap any content in ckeditor tags to turn it into an editable CKEditor. 

<ckeditor id="content">Content</ckeditor>

A unique id must be set for each ckeditor tag in the file it is being used in.

If you wish to store the content in the database you can use ckeditor variables.

{{@content, 'index/content' | ckeditor}}

@content = the content id. 'index/content' is the page / content id. So for example if you had some content on a page called about it might look like;
{{@aboutbox, 'about/aboutbox' | ckeditor}}

If the content does not exist in the database then dummy content will be displayed. Once saved a new row in the database will be created.

How the module works

Inline <ckeditor> tags

The ability to update content directly in a page is available because of inline ckeditor tags. When a user edits and saves the new content it will directly replace the old content. This means that the content always stays within the file and further, allows one to edit the content directly inside the file, duplicate the file or delete the file without worrying about losing reference to the content.

Template code

Template Variable Example
<ckeditor id="content" type="editor type">Content</ckeditor>
{{@contentID, 'page/contentID', 'type of editor' | ckeditor}}
Editor types
type='full' Contains all features of ckeditor. If no type is given, full is assumed.
type='header' Removed images, tables, lists from editor.
type='image' Only has the image button.
